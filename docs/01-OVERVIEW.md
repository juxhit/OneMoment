# 01 — Обзор продукта

## Назначение

OneMoment помогает собирать фотографии гостей на мероприятии без установки приложений:

1. Организатор создаёт событие и получает ссылку / QR-код.
2. Гости открывают страницу на телефоне и загружают снимки.
3. Одобренные фото появляются на Live Wall (проектор / ТВ) почти в реальном времени.
4. После мероприятия — экспорт всех фото в ZIP.

## Роли

| Роль | Интерфейс | Возможности |
|------|-----------|-------------|
| Организатор | `/admin/*` | Настройки, модерация, QR, экспорт, смена события |
| Гость | `/e/{token}` | Загрузка, личная галерея своих фото |
| Зрители | `/wall/{token}` | Просмотр Live Wall |

## Основные функции

- Загрузка JPEG, PNG, WebP (до 20 MB на файл)
- Сжатие на клиенте только если файл превышает лимит
- Серверные превью 400px и 1200px
- Модерация: сразу на стену (G1) или через очередь (G2)
- Опциональный PIN для доступа гостей
- Подпись гостя (`guest_label`) на фото
- Персональная галерея: гость видит только свои загрузки; стена — все одобренные
- Несколько мероприятий: архивирование и создание нового события
- 7 режимов Live Wall: masonry, dynamic_mosaic, carousel, polaroid, generations, global_mosaic, honeycomb
- PWA: «Добавить на экран» на iOS/Android
- Экспорт ZIP с оригиналами и превью

## Ограничения

- Только изображения (видео не поддерживаются)
- Один активный event одновременно для гостей; архивные доступны организатору
- Real-time через SSE с опросом каждые ~2 сек (не WebSocket)
- Рекомендуется HTTPS (обязательно для камеры в Safari на iPhone)

## Типичный сценарий

1. Деплой на VPS, настройка `.env`.
2. Вход в админку, настройка названия и оформления.
3. Страница «Настройка» — QR и ссылка для гостей.
4. Вывод `/wall/{token}` на проектор.
5. При необходимости — включение модерации.
6. После события — экспорт ZIP, архивирование события.
